# performance-db / 実績DB連携

## 概要
複数のスプレッドシートに分散した実績データを収集・変換し、実績データベース用のスプレッドシートに統合するGoogle Apps Scriptツール。台帳データの自動更新やソアスクとの連携処理を含み、クロススプレッドシート間のデータ転送を自動化する。

## 技術構成
| 項目 | 詳細 |
|------|------|
| Platform | Google Apps Script |
| 言語 | JavaScript (GAS) |
| APIs | Google Sheets（SpreadsheetApp） |
| 機能 | クロススプレッドシート転送、データ変換 |
| トリガー | 時間主導型 / 手動実行 |

## ファイル構成
- `実績DB用GAS.js` - 実績データベースへのデータ投入処理
- `台帳元データ更新用GAS.js` - 台帳の元データを更新する処理
- `ソアスク+台帳データ` - ソアスクと台帳データの連携定義
- `台帳Query整理` - 台帳データ取得用クエリの整理
- `台帳自動連携_2` - 台帳データの自動連携処理（第2版）

## セットアップ
1. Google Apps Scriptプロジェクトを新規作成
2. 対象スプレッドシートのIDをスクリプト内に設定
3. 各ソース（ソアスク、台帳等）へのアクセス権限を確認
4. `実績DB用GAS.js` および `台帳元データ更新用GAS.js` をエディタに貼り付け
5. 必要に応じて時間主導型トリガーを設定

## 使い方
- 実績DB更新: `実績DB用GAS.js` の主関数を実行し、各ソースから実績DBへデータを転送
- 台帳更新: `台帳元データ更新用GAS.js` を実行し、台帳の元データを最新化
- 自動連携が設定済みの場合、トリガーにより定期的にデータが同期される
