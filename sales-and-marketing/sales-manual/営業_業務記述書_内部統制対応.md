1. 目的・適用範囲
- 目的：本書は、営業プロセス（商談発生から成約・引継ぎ・知識化まで）の標準業務と内部統制（J‑SOX）に対応したコントロールを定義し、業務の有効性・効率性・財務報告の信頼性・法令遵守を担保することを目的とする。
- 適用範囲：国内営業組織全体（Inside Sales / Field Sales / Solution Sales）。関連部門（SE、営業企画、経理、法務、テクニカルサポート、導入グループ）を含む。
- 適用外：特命案件やM&A由来の特殊取引は「特別案件管理基準」に従う。

2. 関連規程・参照文書
- 企業行動規範、権限規程、稟議規程、与信管理規程、売上計上基準、記録・保存ポリシー、情報セキュリティ基本方針、IT統制基準（アクセス管理、変更管理、運用管理）
- テンプレート群：議事録テンプレ、REDレビューChecklist、見積・粗利計算書、稟議書、顧客提出前Checklist、引継ぎChecklist、Lessons Learned雛形

3. 用語定義（抜粋）
- 案件（Opportunity）：CRM上で管理される商談単位。自動採番（例：OPP-YYYYMMDD-XXXX）。
- REDレビュー：Review‑Edit‑Decisionの略。提案の品質とリスクを短時間でレビューし意思決定する会議。
- 重要統制（Key Control, KC）：財務影響が重大な誤謬を予防・検出するために設計された統制。
- 証跡（Evidence）：統制実施の事実を裏付ける記録（システムログ、承認履歴、添付資料等）。

4. 役割と責任（RACI）
主要活動 × 役割：営業担当（Sales）、SE、上長（Mgr）、営業企画/SalesOps（Ops）、経理（Fin）、法務（Legal）、PM、テクサポ（TS）
- 案件登録：R=Sales / A=Mgr / C=Ops / I=SE,Fin
- 初動対応（顧客連絡・次回設定）：R=Sales / A=Mgr / C=SE / I=Ops
- 詳細ヒアリング・議事録作成：R=Sales / C=SE / A=Mgr / I=Ops
- 提案骨子作成：R=Sales,SE / A=Mgr / C=Ops / I=Fin
- REDレビュー：R=Sales / A=Mgr / C=SE,Fin / I=Ops
- 見積・粗利シミュレーション：R=Sales / C=Fin / A=Mgr / I=Ops
- 稟議起案・承認：R=Sales / A=承認者（Mgr,Fin,Legal）/ C=Fin,Legal / I=Ops
- 顧客提出版チェック：R=Sales / C=Legal,Fin / A=Mgr / I=Ops
- 契約締結：R=Sales / A=Legal / C=Fin / I=Mgr
- 引継ぎ（MC/TSへ）：R=Sales / C=PM,TS / A=Mgr / I=Ops
- ナレッジ登録：R=Sales / C=Ops / A=Mgr / I=全社

5. 使用システム・データ
- CRM/案件管理：案件登録、活動履歴、議事録URL、見積・受注情報
- 稟議ワークフロー：承認ルート自動判定、承認履歴、添付
- ドキュメント管理：提案書、議事録、チェックリスト、契約書ドラフト
- 見積・粗利ツール：価格表、割引ルール、粗利計算
- 契約管理：契約テンプレ、履歴、改定管理
- コラボレーション：メール、会議ツール、Slack 等
- アクセス：ロールベース（最小権限）。四半期レビュー実施。

6. 業務フロー概要
[商談発生] → [案件登録/初動] → [顧客理解・提案骨子] → [REDレビュー] → [見積/粗利] → [稟議・承認] → [顧客提案/交渉] → [受注/契約] → [引継ぎ/納品フォロー] → [知識化]

7. サブプロセス別 業務記述と統制
7.1 商談発生・案件登録（Day0）
- 目的：商談を標準プロセスに乗せ、初動スピードと記録の完全性を担保。
- トリガー：リード創出、紹介、問い合わせ等。
- 入力：顧客情報、商談概要、期待受注額、製品/サービス。
- 実施手順：
  1) CRMで案件登録（必須項目：顧客、案件名、金額概算、確度、担当）
  2) 案件番号の取得と専用フォルダ生成確認
  3) 高難易度判定時はキックオフ招集、SE工数仮押さえ
  4) 顧客へのお礼・次回候補提示・ヒアリングシート送付
- 出力/証跡：CRM案件レコード、活動履歴、メール送信ログ、フォルダ作成ログ
- 主要リスク：未登録/記録漏れ、初動遅延、案件属性の誤り
- コントロール：
  KC K-01 案件登録の完全性（予防/検出）
   内容：CRM必須項目と自動採番、週次未登録リードレポートをOpsがレビュー。
   頻度：リアルタイム/週次、オーナー：Ops、証跡：ダッシュボード、レビュー記録。
  KC K-02 初動対応SLA（予防/検出）
   内容：48時間以内の初回活動必須。SLA違反は上長へ自動通知。
   頻度：日次、オーナー：Mgr、証跡：アラートログ、活動履歴。

7.2 顧客理解・提案骨子（Day1‑3）
- 目的：真因に基づく勝ち筋の明確化。
- 入力：ヒアリングシート、業界/競合調査、SE所見。
- 実施手順：詳細ヒアリング、議事録作成（1時間以内共有）、競合分析、提案骨子/概算見積、勝率評価。
- 出力/証跡：議事録、競合マトリクス、提案骨子、概算見積。
- リスク：顧客要件の誤解、想定外のコスト、過度な値引き前提。
- コントロール：
  KC K-03 議事録の品質とタイムリー性（予防/検出）
   内容：テンプレ準拠、24時間以内共有、上長確認を活動ログで証跡化。
   頻度：都度、オーナー：Sales、証跡：議事録ファイル、共有履歴。

7.3 提案書作成/REDレビュー（Day4‑6）
- 目的：提案品質とリスクの早期摘出。
- 入力：提案ドラフト（完成度60%）、競合分析、概算見積、想定問答。
- 実施手順：30分タイムボックスでRED、改善点整理、ToDo確定。
- 出力/証跡：RED議事、改善ToDo、チェックリスト。
- リスク：重要論点の見落とし、根拠不足。
- コントロール：
  KC K-04 REDレビュー実施統制（予防）
   内容：顧客提案前にRED必須。開催記録とチェックリスト添付を稟議要件に連動。
   頻度：都度、オーナー：Mgr、証跡：会議記録、Checklist。

7.4 見積・粗利シミュレーション（Day4‑6）
- 目的：価格妥当性と収益性の確保。
- 入力：価格表、原価見積、割引ルール。
- 実施手順：粗利シート作成、値引き閾値判定、経理レビュー。
- 出力/証跡：粗利計算書、レビュー記録。
- リスク：粗利過少、誤計算、未承認の値引き。
- コントロール：
  KC K-05 粗利計算とレビュー（予防/検出）
   内容：標準シート使用、経理レビュー必須、数式ロック。
   頻度：都度、オーナー：Fin、証跡：レビューサイン、ファイル版管理。

7.5 稟議・承認（Day7‑9）
- 目的：権限規程に沿った意思決定と責任分担。
- 入力：稟議書（案件概要、利益率、リスク、添付）。
- 実施手順：承認ルート自動判定、根回し、停滞時フォロー。
- 出力/証跡：承認履歴、コメント履歴、添付一式。
- リスク：無権限承認、承認抜け、秘密情報漏えい。
- コントロール：
  KC K-06 稟議承認と職務分掌（予防/検出）
   内容：起案者と承認者の分離、金額/粗利での多段承認、法務/経理の必須承認。
   頻度：都度、オーナー：Mgr,Fin,Legal、証跡：ワークフロー履歴。

7.6 顧客提出・交渉（Day10+）
- 目的：社外秘情報の適切管理と提案の整合性維持。
- 入力：社内版提案書、チェックリスト。
- 実施手順：社内情報の削除、表紙/有効期限/提出日の整合、デモ、フォロー。
- 出力/証跡：顧客提出版、提出記録、宿題管理表。
- リスク：社外秘露出、約束不履行。
- コントロール：
  KC K-07 顧客提出前チェック（予防）
   内容：除外項目（原価/利益/競合分析/内部体制詳細）チェックリスト完了を必須化。
   頻度：都度、オーナー：Sales、証跡：Checklist、最終版ファイル。

7.7 受注・契約締結
- 目的：契約条件の適正と会計処理の適格性。
- 入力：合意条件、契約テンプレ/特約。
- 実施手順：法務レビュー、二者承認、版管理。
- 出力/証跡：締結済契約、法務承認記録、改定履歴。
- リスク：不利条項、コンプラ違反、版ズレ。
- コントロール：
  KC K-08 契約レビューと承認（予防）
   内容：法務レビュー必須、例外は例外承認書で管理。
   頻度：都度、オーナー：Legal、証跡：レビューログ、締結記録。

7.8 引継ぎ・納品フォロー
- 目的：約束事項の確実な遂行と満足度維持。
- 入力：引継ぎパック（提案、議事録、約束事項、リスク、体制）。
- 実施手順：引継ぎ会議、役割分担、当面のアクション設定。
- 出力/証跡：引継ぎ議事録、チェックリスト、フォロー記録。
- リスク：要件伝達漏れ、対応遅延。
- コントロール：
  KC K-09 引継ぎ完了統制（検出）
   内容：引継ぎChecklistの100%完了を受注クローズの条件に連動。
   頻度：都度、オーナー：PM,Sales、証跡：Checklist、会議記録。

7.9 知識化（Lessons Learned / Sales Play）
- 目的：勝ち筋の再現性向上。
- 入力：案件結果、成功/課題、改善案。
- 実施手順：月次レビュー、テンプレ修正、横展開。
- 出力/証跡：成功事例、提案テンプレ更新、DB登録記録。
- リスク：学習機会の逸失、属人化継続。
- コントロール：
  KC K-10 ナレッジ登録の完全性（検出）
   内容：成約後30日以内の登録率をOpsがモニタ、未登録は上長へエスカレーション。
   頻度：月次、オーナー：Ops、証跡：登録率レポート。

8. 職務分掌（SoD）とアクセス統制
- SoD原則：起案と承認の分離、見積と承認の分離、契約ドラフトと法務承認の分離。
- アクセス：ロールベース付与（最小権限）、入退社/異動時の即時反映、四半期アクセスレビュー。
- 統制：
  KC K-11 アクセス権限レビュー（検出）
   内容：四半期ごとに各システムのロール/ユーザ配賦をMgr/システム管理者が照合。
   頻度：四半期、証跡：レビュー台帳、承認記録。

9. 変更管理（テンプレ/価格/承認ルート）
- 文書・ツール・マスタの変更は変更申請→レビュー→承認→周知→施行。
- 統制：
  KC K-12 変更管理統制（予防/検出）
   内容：変更記録と承認履歴の保存、版数管理、施行日明記。
   頻度：都度、オーナー：Ops,Legal,Fin、証跡：変更台帳、版管理履歴。

10. 例外管理
- 例外は「例外申請書」で起案、承認者がリスク評価を付記、例外台帳で追跡、期限を設定。
- 統制：
  KC K-13 例外台帳管理（検出）
   内容：月次で未クローズ例外をレビュー、期限延長は再承認必須。
   頻度：月次、オーナー：Ops、証跡：例外台帳、承認履歴。

11. 証跡と保存期間
- 保存対象：CRM活動履歴、議事録、チェックリスト、見積/粗利計算、稟議承認履歴、提出前ファイル、契約、引継ぎ記録、ナレッジ登録記録。
- 保存期間：原則7年（契約は契約期間＋7年）。
- 保存場所：システム原本（CRM/稟議/契約管理）優先、補助としてドキュメント管理。
- 統制：
  KC K-14 証跡保全（予防/検出）
   内容：自動ログの保持設定、退職/異動時の共有ドライブ引継ぎ、削除保護設定の年次点検。
   頻度：半期、オーナー：情報システム/Ops、証跡：設定スクリーンショット、点検記録。

12. モニタリング（内部統制）
- 月次モニタ：
  1) K-01~K-03 実施率ダッシュボード、SLA違反件数
  2) K-05 粗利レビュー実施率、値引き閾値逸脱件数
  3) K-06 稟議承認のSoD逸脱検知
  4) K-07 提出前チェック完了率
  5) K-09 引継ぎChecklist完了率
  6) K-10 ナレッジ登録率
- 四半期モニタ：K-11 アクセスレビュー、K-12 変更管理抜取監査。
- 年次モニタ：K-14 証跡保全点検、プロセス刷新レビュー。
- 統制：
  KC K-15 監視と是正（検出/是正）
   内容：閾値超過は是正計画（CAP）を起案、翌月フォロー、再発防止策の反映。
   オーナー：Ops/内部統制、証跡：CAP台帳、フォロー会議議事。

13. KPI/品質指標（参考）
