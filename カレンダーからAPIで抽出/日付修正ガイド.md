# 日付エラーの解決方法

## エラーの原因
Google スプレッドシートは日付を自動的に「日付オブジェクト」に変換してしまうため、`Mon Sep 01 2025 01:00:00 GMT+0900` のような形式になってエラーが発生しています。

## 解決方法

### 方法1: Setup Sheetsを再実行（推奨）
1. 「Calendar Export」→「Setup Sheets」を実行
2. Configシートが再作成され、日付が文字列形式で設定されます
3. 日付セルが自動的に「書式なしテキスト」に設定されます

### 方法2: 手動で日付を修正
1. Configシートを開く
2. START_DATEとEND_DATEのセル（B2とB3）を選択
3. 右クリック→「表示形式」→「書式なしテキスト」を選択
4. 日付を以下の形式で入力：
   - START_DATE: `2025-09-01`
   - END_DATE: `2025-09-30`

### 方法3: 日付の前にアポストロフィを付ける
1. Configシートを開く
2. 日付を入力する際、先頭に `'` を付ける
   - START_DATE: `'2025-09-01`
   - END_DATE: `'2025-09-30`
3. アポストロフィは表示されませんが、文字列として扱われます

## 重要なポイント
- **必ずYYYY-MM-DD形式**で入力（例：2025-09-01）
- スラッシュ（/）やピリオド（.）は使用しない
- 日付セルの書式は「書式なしテキスト」に設定

## コード改善内容
最新のコードでは以下の改善を実施：
1. 日付オブジェクトを自動的にYYYY-MM-DD形式に変換
2. setupSheets実行時に日付セルを文字列形式に設定
3. より詳細なエラーメッセージとログ出力

## テスト方法
1. Configシートで日付を正しく設定
2. 「Calendar Export」→「Test Connection」でAPI接続確認
3. 「Calendar Export」→「Run Export」を実行

これで正常に動作するはずです。
